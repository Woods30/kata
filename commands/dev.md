# Kata Dev

Run the Kata development workflow.

## What To Do

1. Read `skills/kata/workflows/development.md` if available.
2. Use installed development skills first, such as TDD, subagent, worktree, or review skills.
3. Execute the next task from `docs/todo.md` or `docs/prompt_plan.md`.
4. Keep tests, `docs/todo.md`, and `.kata/memory/session.md` current.

## Equivalent CLI

```bash
kata dev
```
