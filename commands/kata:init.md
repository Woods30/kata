# Kata Init

Initialize the current project for Kata.

## What To Do

Run the Kata init workflow:

1. Read `skills/kata/workflows/init.md` if available.
2. Create or update project instructions, docs, shared memory, hook settings, and skill inventory.
3. Preserve existing files and report created, skipped, and manually-merge-needed files.

## Equivalent CLI

```bash
kata init
```

This command exists so users can run `/kata:init` directly without knowing the subcommand form `/kata init`.
